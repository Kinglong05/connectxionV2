# ConnectXion Gaming Hub - Deployment Guide

This repository contains the full source code for the ConnectXion Gaming platform, prepared for production.

## 🛠️ Components
- **Client (PHP/MySQL)**: The main web interface.
- **Server (Node.js/Socket.IO)**: The real-time signaling engine.

---

## 🚀 1. Deploy the Signaling Server (Node.js)
The signaling server handles real-time messaging, voice, and push notifications.

1.  Create an account on [Render.com](https://render.com/).
2.  Click **New +** > **Web Service**.
3.  Connect this GitHub repository.
4.  Set the **Root Directory** to `server/`.
5.  Render will automatically detect the settings from `render.yaml`.
6.  **Environment Variables**: Add your production Database credentials (`DB_HOST`, `DB_USER`, etc.) in the Render dashboard.

---

## 🌐 2. Deploy the Web Interface (PHP)
1.  Upload the contents of the `client/` folder to your PHP web host (e.g., Hostinger, InfinityFree).
2.  **Database Setup**:
    -   Create a new MySQL database on your host.
    -   Import the `connectxion.sql` file provided in this repo.
3.  **Configuration**:
    -   Update the database credentials in `db.php` if you are not using environment variables.

---

## 📱 3. Enable PWA & Push Notifications
1.  Ensure your PHP host is using **HTTPS**.
2.  Open your site on your phone.
3.  Browser will prompt you to "Add to Home Screen".
4.  Grant "Notification" permissions when prompted.

---

## 🔒 4. Production Security Tips
- Change the `ALLOWED_ORIGINS` in `server/render.yaml` from `*` to your actual PHP domain.
- Ensure your `.env` files are never uploaded to the public `client/` directory.
